## Setup

To access OCI Generative AI models, you'll need an OCI account with access to the Generative AI service, proper authentication configured, and the `langchain-oci` integration package.

### Credentials

OCI supports four authentication methods following the [standard OCI SDK authentication](https://docs.oracle.com/en-us/iaas/Content/API/Concepts/sdk_authentication_methods.htm):

1. **API Key** - Uses credentials from `~/.oci/config` (default)
2. **Security Token** - Session-based authentication
3. **Instance Principal** - For OCI compute instances
4. **Resource Principal** - For OCI Functions and other resources

To set up API key authentication (recommended for getting started):

```bash
oci setup config
```

This creates `~/.oci/config` with your credentials in the `DEFAULT` profile.

### Installation

The LangChain OCI integration lives in the `langchain-oci` package. You'll also need the `oci` SDK:

<CodeGroup>
```bash pip
pip install -qU langchain-oci oci
```

```bash uv
uv add langchain-oci oci
```
</CodeGroup>

## Instantiation

Now we can instantiate our model object and generate chat completions:

```python
from langchain_oci import ChatOCIGenAI

llm = ChatOCIGenAI(
    model_id="meta.llama-3.3-70b-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..your-compartment-id",
    # Optional parameters:
    # auth_type="API_KEY",  # Default
    # auth_profile="DEFAULT",  # Profile name in ~/.oci/config
    # model_kwargs={"temperature": 0.7, "max_tokens": 500},
)
```

## Streaming

Stream responses token-by-token for real-time output:

```python
for chunk in llm.stream(messages):
    print(chunk.content, end="", flush=True)
```

```text
J'adore la programmation.
```

### Async Streaming

```python
async for chunk in llm.astream(messages):
    print(chunk.content, end="", flush=True)
```

## Async

All operations support async for production workloads:

```python
import asyncio

async def main():
    # Single async request
    response = await llm.ainvoke(messages)
    print(response.content)

    # Concurrent requests
    results = await asyncio.gather(
        llm.ainvoke("What is 2+2?"),
        llm.ainvoke("What is 3+3?"),
        llm.ainvoke("What is 4+4?"),
    )
    for r in results:
        print(r.content)

asyncio.run(main())
```

## Tool Calling

Bind tools to enable function calling:

```python
from langchain.tools import tool

@tool
def get_weather(city: str) -> str:
    """Get the weather for a city."""
    return f"Weather in {city}: 72°F, sunny"

@tool
def get_time(timezone: str) -> str:
    """Get current time in a timezone."""
    return f"Time in {timezone}: 2:30 PM"

llm_with_tools = llm.bind_tools([get_weather, get_time])
response = llm_with_tools.invoke("What's the weather in Chicago?")

# Access tool calls
print(response.tool_calls)
```

```python
[{'name': 'get_weather', 'args': {'city': 'Chicago'}, 'id': 'call_1'}]
```

### Parallel Tool Calls

Llama 4+ models support parallel tool execution:

```python
from langchain_oci import ChatOCIGenAI

llm = ChatOCIGenAI(
    model_id="meta.llama-4-scout-17b-16e-instruct",  # Llama 4+ required
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..xxx",
)

llm_with_tools = llm.bind_tools(
    [get_weather, get_time],
    parallel_tool_calls=True,  # Enable parallel execution
)

response = llm_with_tools.invoke("What's the weather in Chicago and the time in New York?")
print(len(response.tool_calls))  # 2 - both tools called in parallel
```

### Tool Configuration

```python
llm_with_tools = llm.bind_tools(
    tools=[get_weather],
    tool_choice="auto",  # "auto", "required", "none", or specific tool name
    parallel_tool_calls=False,  # Enable parallel execution (Llama 4+)
    max_sequential_tool_calls=8,  # Limit consecutive calls
    tool_result_guidance=False,  # Guide model to use results naturally
)
```

## Structured Output

Extract structured data using Pydantic models:

```python
from pydantic import BaseModel, Field

class Contact(BaseModel):
    """Contact information."""
    name: str = Field(description="Person's full name")
    email: str = Field(description="Email address")
    phone: str = Field(description="Phone number")

structured_llm = llm.with_structured_output(Contact)

result = structured_llm.invoke(
    "Extract contact info: John Doe, email john@example.com, phone 555-1234"
)

print(f"Name: {result.name}")
print(f"Email: {result.email}")
print(f"Phone: {result.phone}")
```

```text
Name: John Doe
Email: john@example.com
Phone: 555-1234
```

### Structured Output Methods

```python
# Method 1: Function calling (most reliable, default)
structured_llm = llm.with_structured_output(Contact, method="function_calling")

# Method 2: JSON mode (for simple schemas)
structured_llm = llm.with_structured_output(Contact, method="json_mode")

# Method 3: JSON schema (native OCI support)
structured_llm = llm.with_structured_output(Contact, method="json_schema")
```

### Image Analysis

```python
from langchain.messages import HumanMessage
from langchain_oci import ChatOCIGenAI, load_image

llm = ChatOCIGenAI(
    model_id="meta.llama-3.2-90b-vision-instruct",

message = HumanMessage(

        {"type": "text", "text": "What's in this image? Describe in detail."},
        load_image("./photo.jpg"),  # Or use URL
    ]
)

response = llm.invoke([message])
print(response.content)
```

### Vision Utility Functions

```python
from langchain_oci import load_image, encode_image, to_data_uri, is_vision_model

# Load image from file or URL
image_block = load_image("./photo.jpg")
image_block = load_image("https://example.com/image.png")

# Encode raw bytes
with open("photo.jpg", "rb") as f:
    image_block = encode_image(f.read(), "image/jpeg")

# Convert to data URI
data_uri = to_data_uri(image_block)

# Check if model supports vision
if is_vision_model("meta.llama-3.2-90b-vision-instruct"):
    print("This model supports vision!")
```

## Gemini Multimodal (PDF, Video, Audio)

Google Gemini models support PDF documents, video, and audio files:

### PDF Processing

```python
import base64
from langchain.messages import HumanMessage
from langchain_oci import ChatOCIGenAI

llm = ChatOCIGenAI(
    model_id="google.gemini-2.5-flash",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..xxx",
)

# Load and encode PDF
with open("document.pdf", "rb") as f:
    pdf_data = base64.b64encode(f.read()).decode()

message = HumanMessage(content=[
    {"type": "text", "text": "Summarize this PDF document"},
    {"type": "media", "data": pdf_data, "mime_type": "application/pdf"}
])

response = llm.invoke([message])
print(response.content)
```

## Model Configuration

Configure model behavior with `model_kwargs`:

```python
llm = ChatOCIGenAI(
    model_id="meta.llama-3.3-70b-instruct",
    service_endpoint="https://inference.generativeai.us-chicago-1.oci.oraclecloud.com",
    compartment_id="ocid1.compartment.oc1..xxx",
    model_kwargs={
        "temperature": 0.7,      # Creativity (0.0-1.0)
        "max_tokens": 500,       # Max response length
        "top_p": 0.9,           # Nucleus sampling
        "top_k": 50,            # Top-k sampling
        "frequency_penalty": 0.0,  # Reduce repetition
        "presence_penalty": 0.0,   # Encourage new topics
    },
)
```

## Error Handling

```python
from oci.exceptions import ServiceError

try:
    response = llm.invoke("Hello!")
except ServiceError as e:
    if e.status == 401:
        print("Authentication failed - check your credentials")
    elif e.status == 404:
        print("Model or endpoint not found")
    elif e.status == 429:
        print("Rate limit exceeded")
    else:
        print(f"Error: {e.message}")
```
